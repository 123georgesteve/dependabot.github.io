Rely on everyone else's test suites

Dependabot aggregates everyone's test suite results to check a dependency update is backwards compatible.

How it's calculated

1

We update hundreds of apps

When a new dependency version is released, Dependabot creates similar pull requests for hundreds of repos.

2

We listen for the test results

For each repo with CI enabled and a previously passing test suite, we learn whether the update breaks any tests.

3

We show you the pass rate

Our compatibility score is the % of CI runs that passed when updating from your current version to the new one.

How it can help you

Check backwards compatibility

Not all dependency authors use semantic versioning. Not all dependencies are post v1.0.0. When you can't rely on SemVer, checking everyone's tests pass is the next best thing.

Avoid buggy releases

Despite everyone's best intentions, bugs sometimes creep into new releases. When you know that everyone's tests pass, however, you can be relatively sure an update is bug-free.
